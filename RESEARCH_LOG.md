# Research log — <your-project-name>

Canonical, append-only record of what the project is, what has been agreed, and what has been rejected. Driven by the user, not the agent.

`IDEAS_TRAIL.md` is the agent-maintained narrative. `research_log.md` is the decision record. They are separate on purpose.

## Agent identity

The user identifies each agent as **Agent A**, **Agent B**, or **Agent C** per session. Do **not** identify yourself by model name (Claude, GPT, Gemini, etc.) anywhere in this repo. Ideas are evaluated on merit, not on model reputation.

If the user has not told you which letter you are this session, ask before writing here.

## Reasoning effort

You cannot reliably introspect your own reasoning effort — it is set by the harness. **Before appending, ask the user what reasoning effort to record** (e.g. Low / Medium / High / Max). Record their answer verbatim in the appendix header.

## How to append

1. Confirm your agent letter (A/B/C) and ask for the current reasoning effort.
2. Append a new section at the bottom. Never edit prior sections. Heading format:

   ```
   ## Appendix <letter> — <short topic>

   *Appended <YYYY-MM-DD HH:MM TZ> by Agent <letter>. Reasoning strength: <user-provided>.*
   ```

3. If the user agrees or disagrees with specific points, record both sides. The file reflects the actual decision, not just your view.
4. Be honest. If something is BS, say so.
